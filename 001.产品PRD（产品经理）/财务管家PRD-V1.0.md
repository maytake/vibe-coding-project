## 1. 产品概述

### 1.1 产品背景

在日常生活中，个人和家庭往往缺乏系统性的理财记录工具。大多数人通过微信支付、支付宝、银行APP查看消费记录，但这些工具分散且不直观，难以对消费行为进行有效分析和规划。小微企业和个体户更需要清晰的收支记录来了解经营状况。

### 1.2 产品定位

**财务管家** 是一款轻量级的个人/家庭财务管理工具，帮助用户快速记录日常消费/收入，提供直观的统计分析和财务报表，让用户清晰地了解钱花在哪里、赚了多少。

### 1.3 核心价值主张

- **简单高效**：3秒内完成一笔记账，告别繁琐流程
- **清晰可见**：消费分类统计，每月收支一目了然
- **多人协作**：支持家庭/情侣共享账本，共同管理财务

## 2. 目标用户与场景

### 2.1 主要用户画像

| 画像 | 用户A（个人） | 用户B（家庭） | 用户C（小微企业主） |
|------|--------------|--------------|-----------------|
| 年龄 | 25-35岁 | 30-45岁 | 28-50岁 |
| 收入方式 | 工资为主 | 工资+副业 | 经营收入 |
| 使用场景 | 记录日常消费，控制支出 | 家庭共同记账，预算管理 | 记录进货、营收、成本 |

| 期望 | 简单、快速看到分析 | 两人共享，各自记录 | 支持自定义分类、打印导出 |

### 2.2 第一版核心用户

**优先级最高**：个人用户（18-35岁），记录日常消费，控制开支  
**优先级次高**：家庭/情侣，共同管理日常收支  
**优先级兼顾**：小微企业和个体户，记录简单收支

## 3. 功能需求

### 3.1 核心功能清单

| 功能模块 | 功能点 | 优先级 | 描述 |
|---------|--------|--------|------|
| **记账** | 快速记账 | P0 | 选择分类→输入金额→保存，3步完成 |
| **记账** | 编辑/删除记录 | P0 | 修改或删除已记账目 |
| **记账** | 选择日期 | P1 | 支持补记账，可选择历史日期 |
| **记账** | 添加备注 | P1 | 每笔记录可添加备注说明 |
| **账户** | 多账户管理 | P1 | 现金、微信、支付宝、银行卡等 |
| **分类** | 默认分类 | P0 | 餐饮、交通、购物、娱乐等预设分类 |
| **分类** | 自定义分类 | P1 | 用户可添加自定义收支分类 |
| **统计** | 月度收支报表 | P0 | 本月收入、支出、结余 |
| **统计** | 分类占比 | P0 | 各分类支出占比（饼图） |
| **统计** | 趋势分析 | P1 | 近几个月收支趋势（折线图） |
| **账本** | 多账本 | P2 | 家庭账本、装修账本等 |
| **数据** | 导出Excel | P1 | 导出为Excel文件 |

### 3.2 详细功能描述

#### 3.2.1 快速记账

- **入口**：首页大按钮「+」，点击直接进入记账页面
- **流程**：
  1. 选择收支类型（支出/收入）
  2. 选择分类（图标+名称）
  3. 输入金额（数字键盘）
  4. 可选：选择账户、添加备注、选择日期
  5. 点击保存
- **反馈**：保存成功 toast 提示，返回首页列表自动更新

#### 3.2.2 分类体系

**支出默认分类**（10个）：
- 🍚 餐饮
- 🚗 交通
- 🛒 购物
- 🎬 娱乐
- 🏠 住房
- 💊 医疗
- 📱 通讯
- 👔 服饰
- 🎁 人情
- 📦 其他

**收入默认分类**（5个）：
- 💰 工资
- 💵 兼职
- 🎯 奖金
- 🎁 礼金
- 📥 其他

#### 3.2.3 账户管理

支持的账户类型：
- 💵 现金
- 💳 银行卡
- 📱 微信支付
- 🟢 支付宝
- 📘 银行账户

每个账户可设置：
- 账户名称（可自定义）
- 账户类型
- 初始余额
- 颜色/图标

#### 3.2.4 统计报表

**月度概览**：
- 本月收入总计
- 本月支出总计
- 本月结余（收入-支出）
- 与上月对比（上升/下降百分比）

**收支明细**：
- 按日期排列的记账列表
- 支持筛选（按分类、按账户、按日期范围）

**分类统计**：
- 饼图展示各分类占比
- 点击分类可查看该分类下的详细记录

#### 3.2.5 数据导出

- 导出格式：Excel（.xlsx）
- 导出内容：日期、类型、分类、账户、金额、备注
- 筛选能力：可选择日期范围

## 4. 非功能需求

### 4.1 性能要求

- 页面首次加载 ≤ 2秒
- 记账保存响应 ≤ 500ms
- 支持10000条记账数据，查询不卡顿

### 4.2 兼容性

- **浏览器**：Chrome、Safari、Edge 最新版
- **手机浏览器**：iOS Safari、Android Chrome 及微信内置浏览器
- **屏幕适配**：响应式布局，支持手机竖屏（375px-428px）和平板横屏

### 4.3 安全要求

- 用户数据本地存储（浏览器localStorage）+ 云端同步（后续版本）
- 敏感数据不记录身份证号、银行卡号等
- 登出后清除本地会话标识

### 4.4 辅助功能

- 深色模式支持（跟随系统设置）
- 无障碍支持：按钮有足够的点击区域（至少44x44px）

### 6.3 配色方案

| 用途 | 颜色 | 色值 |
|------|------|------|
| 主色 | 樱花粉 | #FFB7C5 |
| 主色（深） | 玫瑰粉 | #FF8FA3 |
| 收入 | 薄荷绿 | #7DD3C0 |
| 支出 | 珊瑚红 | #FFA09B |
| 背景 | 奶油白 | #FFF9F5 |
| 卡片背景 | 浅粉白 | #FFF0F3 |
| 文字主 | 暖灰色 | #4A4A4A |
| 文字次 | 浅灰色 | #8E8E8E |
| 高亮强调 | 薰衣草紫 | #C9B8E8 |
| 按钮渐变 | 粉紫渐变 | #FFB7C5 → #C9B8E8 |

> **配色说明**：
> - 主色调采用樱花粉（#FFB7C5），柔和可爱，适合追求温馨感的用户
> - 收入用薄荷绿，清新舒适；支出用珊瑚红，温和不刺眼
> - 卡片使用奶油白底+浅粉边框，营造温暖氛围
> - 整体饱和度偏低，视觉舒适不疲劳

### 6.4 字体

- **主字体**：无衬线字体，系统默认（-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto）
- **数字字体**：等宽数字（Tabular nums），确保金额对齐
- **字号**：
  - 金额大字：32px / bold
  - 列表文字：16px
  - 辅助文字：14px
